# Standards Map

Use standards as supporting context, not filler. Map a finding only when it clarifies impact, remediation, or verification.

## Web/API

- OWASP Secure Coding Practices: input validation, output encoding, authentication, session management, access control, cryptography, logging, data protection, secure communication, configuration, database security, file management, and coding practices.
- OWASP ASVS: authentication, session management, access control, validation, API security, configuration, data protection, and logging.

## AI/LLM

- OWASP GenAI/LLM Top 10 2025: prompt injection, sensitive information disclosure, supply-chain risk, data/model poisoning, improper output handling, excessive agency, system prompt leakage, vector/embedding weaknesses, misinformation, and unbounded consumption.
- Prioritize prompt injection, improper output handling, excessive agency, sensitive information disclosure, supply chain, and unbounded consumption.

## Secure AI Lifecycle

- NIST SSDF/SP 800-218A and NCSC secure AI development guidance: secure design, development, deployment, operation, maintenance, supply-chain risk management, documentation, asset management, technical debt, incident response, and vulnerability handling.

## Supply Chain

- SLSA-style concepts: source integrity, build integrity, dependency provenance, package tampering resistance, CI/CD permissions, reproducibility, and pinned or signed actions where practical.
